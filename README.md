## Процесс работы

Стилизовал уже работающее приложение: дополнял `style.css`, не трогая готовую HTML-разметку и JavaScript. Двигался поэтапно — сначала общий лейаут страницы, затем размеры блоков, типографика, формы, кнопки, состояния карточек и, в конце, позиционирование прелоадеров.

### Стек технологий

- Семантическая вёрстка HTML5 (готовая разметка)
- CSS3
- Flexbox
- CSS logical properties — `inline-size`, `block-size`, `margin-block`, `padding-inline`
- Методология именования БЭМ
- Кастомные элементы форм (псевдочекбоксы поверх скрытых нативных)
- Доступность: `:focus-visible`, `:has()`, паттерн `.visually-hidden`
- Stylelint + Prettier для контроля качества и форматирования

### Чему я научился

- **Раскладке на Flexbox** на нескольких уровнях вложенности: разница между главной и поперечной осями, `justify-content` vs `align-items`, и `align-self` для исключения одного элемента из общего выравнивания.
- **Ловушке `min-width: auto`** у flex-элементов. Долго не мог понять, почему длинный текст не обрезается многоточием — оказалось, flex-элемент по умолчанию не сжимается меньше своего содержимого. Лечится `min-width: 0` на *каждом* уровне flex:
  `min-inline-size: 0;`
- **Работе с медиа** через `object-fit: cover` и `object-position` — изображения и видео заполняют контейнер без искажения пропорций.
- **Обрезке текста** двумя способами: однострочной через `text-overflow: ellipsis` (нужна связка `white-space: nowrap` + `overflow: hidden`) и многострочной через `line-clamp`.
- **Кастомным чекбоксам с нуля**: спрятать нативный `<input>` паттерном `.visually-hidden`, нарисовать свой квадрат и переключать его заливку через `:checked` и псевдоэлемент `::after`.
- **Доступности форм**: осмысленный фокус через `:focus-visible` (обводка только при навигации с клавиатуры) и родительский селектор `:has()`.
- **Паттерну «общий класс + модификаторы»** для кнопок — общее в `.button`, различия в `.search-form__submit-button` и `.more-button`.
- **Связке `position: relative` ↔ `position: absolute`** для наложения прелоадеров точно поверх своих контейнеров.
- **Гигиене кода**: порядок CSS-правил по структуре DOM, понимание специфичности селекторов (`no-descending-specificity`) и настройка Stylelint локально.
